# Research Watch: xAI Grok 4.3

- Repo: https://x.ai/news (no public repo — commercial API)
- Also see: https://docs.x.ai/developers/models | https://openrouter.ai/x-ai/grok-4.3 | https://artificialanalysis.ai/models/grok-4-3 | https://venturebeat.com/technology/xai-launches-grok-4-3-at-an-aggressively-low-price-and-a-new-fast-powerful-voice-cloning-suite

## Why this is worth watching

Grok 4.3 landed April 30, 2026, with a 37–83% price cut versus Grok 4.20 across input/output token tiers, placing it near Mistral Medium 3.5 in blended cost while operating at a higher capability tier by benchmark. The 321-point ELO gain on GDPval-AA (agentic task evaluation) is the largest single-release jump logged in this taxonomy to date, lifting Grok 4.3 above Claude Sonnet 4.6 on that index. Combined, the price and agentic benchmark movements make this a direct pressure signal on clawfit's LLM scoring weights — particularly the cost axis.

## What stands out immediately

- **Context window:** 1,000,000 tokens confirmed (OpenRouter, xAI docs). Not 2M — the 2M figure is Grok 4.1 Fast, a different model; claim-to-inspect on any 2M attribution in secondary sources.
- **Pricing vs. predecessor:** Input $1.25/M (down ~58% from $3.00), output $2.50/M (down ~83% from $15.00); cached input $0.31/M; batch API at 50–80% further discount. Tier-change note: requests exceeding 200k total tokens are billed at a higher rate — no public figure yet for the elevated tier.
- **Agentic benchmark:** GDPval-AA ELO 1500 (up from 1179 in 4.20). Scores 53 on Artificial Analysis Intelligence Index, just above Claude Sonnet 4.6. Trails GPT-5.5 by 276 ELO points on the same agentic index.
- **Always-on reasoning:** Reasoning mode is default and cannot be disabled. ~44% more output tokens than Grok 4.20, which at the new output pricing is still cheaper than 4.20's baseline. Downstream cost impact on agentic loops is claim-to-validate given the higher output token volume.
- **Multimodal:** Native video input (≤5 min, ≤1080p recommended); image input (≤20 MiB); direct PDF/XLSX/PPTX document generation. First xAI model with video input in the API.
- **Server-side tools:** Web search, X search, and code execution each priced at $5/1,000 calls — billed separately from token costs. Not a native function-calling schema in the standard OpenAI tool-call sense; claim-to-inspect for full structured-output conformance.
- **16-agent Heavy system:** Inherited from Grok 4.20. Available only via SuperGrok Heavy subscription ($300/month) — not accessible at the API tier. For clawfit API-profile users, multi-agent orchestration must be implemented at the application layer; the Heavy scheduler is not an API primitive.
- **Voice APIs:** Separate STT/TTS at $4.20/M characters — claimed 86–92% cheaper than comparable tier. Out of scope for the LLM registry entry but relevant to Level 6 tracking.
- **Output speed:** ~207 tokens/second — faster than most reasoning-class models surveyed.

## Why clawfit should care

The LLM registry currently has no xAI entry. Grok 4.3's blended cost (~$1.56/M at 3:1 input:output) now places it between gpt-4o-mini ($0.15) and claude-sonnet ($3.00) on the cost axis while scoring above claude-sonnet on agentic benchmarks. If this holds on independently confirmed evals, the current scoring model's LLM preference weight (0.15) may route code-gen and research tasks away from a legitimately cheaper agentic option. The 2026-04-28 meta-pattern note ("cost weight 0.25 and LLM preference weight 0.15 may both be under-weighted for power-user profiles where vendor switching cost is near-zero") gains a direct data point here.

The always-on reasoning token overhead is a confounding factor: users on `latency: low` profiles would need to absorb ~44% output token inflation at the non-reasoning tier with no opt-out. This is a hard constraint that limits Grok 4.3 to `latency: medium` or `latency: high` profiles.

## Preliminary interpretation

Current best reading:
- **Level 1 — Base runtimes / primary agent surfaces** (LLM substrate used directly or via API by Level 1–3 harnesses)
- Registry classification: LLM provider entry, not an agent or harness — same schema slot as `gpt-4o`, `claude-sonnet`, `mistral-medium-3-5`
- Relevant task labels if added: `qa`, `research`, `code-gen`, `data-analysis` — agentic benchmark results favor `code-gen` and `research`; `latency: medium` given always-on reasoning token inflation; `network: online` only
