# Research Watch: Kiro CLI UX refresh as agent-native terminal signal

- Source: <https://kiro.dev/blog/new-look-for-cli/>
- Product: <https://kiro.dev/>

## Why this is worth watching
This is useful as a research/watch signal because it shows a base agent runtime maturing its terminal experience around:
- unified live status
- omnipresent input
- interruption / redirection
- slash commands and @ context
- contextual overlays
- future support for more complex sub-agent / multi-agent workflows

## Why clawfit should care
Kiro belongs in the broader base-runtime conversation, but this specific post is interesting because it highlights a separate trend:

> terminal agent UX is becoming its own competitive surface.

This means that for agent-native CLIs, differentiation may increasingly come from:
- how steerable the session is
- how interruption works
- how context is injected
- how tool permission flows are presented
- how multi-agent complexity is surfaced without breaking terminal ergonomics

## Preliminary interpretation
- `Kiro CLI` fits as a **Level 1 base runtime / primary agent surface**
- this blog post should be tracked as a **Level 7-adjacent interface evolution signal**
