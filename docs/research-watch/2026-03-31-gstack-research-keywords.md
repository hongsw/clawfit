- Claude Code skill packs

This note records the most useful **next research keyword ideas** and why they matter.

---

## Core reading of gstack
`gstack` is not best understood as a simple prompt pack.
It is closer to a:

> **team-shaped Claude Code skill operating system**

Important features/signals:
- 20+ specialist roles encoded as slash-command skills
- workflow chain: think → plan → build → review → test → ship → reflect
- browser-assisted QA and headed Chrome control
- vendorable project-local skill packs
- cross-host ambitions (Claude Code, Codex, Gemini, Factory Droid)
- built-in retrospectives, telemetry, review logging, safety, and release flow

This makes it an especially strong signal for:
- team harnesses
- executable workflow packs
- skill operating systems
- “virtual engineering team” framing

## High-priority research keyword ideas

### 1. virtual engineering team
Why search it:
- captures the headline framing of gstack
- likely to surface similar systems that package roles rather than just tasks
- useful for differentiating role-based orchestration from generic subagent systems

### 2. Claude Code skill pack / slash-command skills
Why search it:
- gstack is deeply skill-centric
- likely surfaces other repos treating Claude Code as a host for reusable operating systems
- useful for understanding the **skill-pack economy** forming around Claude Code

Suggested variants:
- `Claude Code skill pack`
- `Claude Code slash commands`
- `Claude Code skills repo`
- `SKILL.md standard repos`

### 3. vendorable AI operating system
Why search it:
- gstack’s “copy into the repo and teammates get it” behavior is important
- surfaces systems where AI workflow becomes project infrastructure, not only user-global configuration
- useful for executable SSOT and team reproducibility research

Suggested variants:
- `vendorable skill pack ai`
- `repo-local AI workflow system`
- `project-local ai operating system`

### 4. browser-connected agent workflows
Why search it:
- gstack’s browse / connect-chrome / cookie import / sidepanel / handoff flow is strategically important
- this connects to QA, consumer workflows, authenticated flows, and real-surface automation

Suggested variants:
- `Claude Code browser automation`
- `headed browser ai agent`
- `Chrome sidepanel AI workflow`
- `authenticated browser agent workflow`

### 5. AI software factory / software factory harness
Why search it:
- the repo explicitly frames itself as open-source software factory
- this connects to company orchestration, team harnesses, and repeatable production pipelines
- useful for comparing against Paperclip, DeerFlow, Scion, and revfactory/harness

Suggested variants:
- `AI software factory`
- `software factory agent harness`
- `open source software factory ai`

### 6. retro / review / ship loops as first-class AI workflow
Why search it:
- many repos cover planning or coding, but fewer encode review, retro, and deployment as reusable AI workflow primitives
- useful for understanding where **full sprint-loop packaging** is emerging

Suggested variants:
- `AI retro workflow`
- `AI release engineer workflow`
- `AI code review qa ship loop`

### 7. cross-host skill portability
Why search it:
- gstack explicitly supports multiple hosts beyond Claude Code
- this matters for standardization, interoperability, and portability of AI operating layers

Suggested variants:
- `SKILL.md interoperability`
- `cross-agent skill pack`
- `Claude Code Codex Gemini shared skills`

### 8. AI agent org design / role bundles
Why search it:
- gstack is strongly role-shaped (CEO, Eng Manager, Designer, QA, CSO, Release Engineer)
- useful to compare against company orchestration systems and meta-skill generators

Suggested variants:
- `AI agent org chart`
- `role-based agent team`
- `agent team architect`
- `virtual executive team AI`
