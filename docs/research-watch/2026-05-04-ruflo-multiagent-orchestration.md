# Research Watch: Ruflo (formerly Claude Flow)

- Repo: https://github.com/ruvnet/ruflo
- Also see: https://flo.ruv.io (web UI), https://goal.ruv.io (GOAP planner), release notes at https://github.com/ruvnet/ruflo/releases

## Why this is worth watching
Ruflo hit GitHub Trending #1 on 2026-05-03 with +1,834 stars in a single day, reaching 38,813 total — the highest single-day star velocity of any L2 harness in this taxonomy since Warp's +11,955 on 2026-04-28, though Ruflo's is organic growth on top of a multi-month base rather than a one-day open-source conversion event. The project rebranded from "Claude Flow" to Ruflo in the v3.5 cycle, absorbing 6,136 commits, 1,471 releases, and a claimed 500,000 downloads / 100,000 monthly active users from the earlier name — making this the most deeply maintained Claude Code harness tracked in this taxonomy. The v3 ground-up TypeScript + WASM Rust rewrite, zero-trust multi-machine federation, and SONA self-learning layer represent a feature surface that is structurally broader than any single existing L2 entry.

## What stands out immediately
- Architecture is explicitly layered: Claude Code / Claude API → Orchestration (MCP server, router, 27 hooks) → Swarm Coordination (queen agent, topology, Raft/Byzantine/Gossip consensus) → 100+ Specialized Agents → Memory / Learning (AgentDB, HNSW, SONA, ReasoningBank) → LLM Providers. This is a claim-to-inspect diagram, not an independently verified runtime trace.
- Rebranding from "Claude Flow" is incomplete as of v3.5–v3.6 — open issues (#1254, #1620) confirm the statusline still outputs "Claude Flow V3" and installer scripts still reference the old `claude-flow` CLI command. Treat the rebrand as in-progress, not complete.
- SONA neural learning and ReasoningBank are described in marketing terms ("agents learn from every task, remember across sessions") without published benchmarks or ablation data. Classify as design-intent, not validated fact.
- HNSW vector indexing via AgentDB claims "150x–12,500x faster search" versus brute force — this is a property of HNSW in general, not a Ruflo-specific achievement; the claim does not show what baseline the upper bound targets.
- Zero-trust federation (mTLS + ed25519 challenge-response, 14-type PII detection pipeline, behavioral trust scoring formula `0.4×success + 0.2×uptime + 0.2×threat + 0.2×integrity`) is architecturally coherent; federation implementation is tracked as partially complete in issue #1669 — partial completion status is a material qualifier for enterprise profiles.
- 210 MCP tools across five server groups plus 18 offline browser tools; 32 native plugins; 12 auto-triggered background workers. The quantitative surface area is among the largest of any L2 entry, but also raises composability and audit surface concerns not addressed in the README.
- Tiered LLM routing (Haiku / Sonnet / Opus by task complexity) is stated as a cost-reduction mechanism; the "75–80% reduction in token consumption" claim is consistent with the V3 rebuild rationale but is not independently verified.
- Web UI at flo.ruv.io exposes multi-model routing (Qwen, Claude, Gemini, OpenAI) with parallel MCP tool calling — this extends Ruflo's scope from a Claude-only harness to a multi-provider orchestration surface, which materially affects L2 classification relative to Claude-specific wrappers.
- SWE-Bench Verified 84.8% cited in external sources; the README does not reproduce this figure directly. Treat as unconfirmed until the primary README or a release note cites the benchmark methodology.
- TypeScript 66% / JavaScript 21.5% / Python 7.9% / Shell 2.8% / Rust 0.3% breakdown — the Rust fraction is small relative to the WASM kernel marketing; the policy engine is Rust-compiled to WASM but the dominant codebase surface is TypeScript.

## Why clawfit should care
Ruflo is the largest-star, most-maintained Claude Code harness in the tracked taxonomy, and its feature surface spans four of the seven ecosystem layers simultaneously: harness routing and swarm coordination (L2), SONA self-learning and HNSW memory persistence (L5), 210-tool MCP surface (L4c), and multi-provider LLM federation that approaches a base-runtime concern (L1 adjacent). That multi-layer collapse is relevant to clawfit's scoring model because a team adopting Ruflo is not just selecting a harness — they are effectively selecting a memory substrate, a tool registry, and an LLM routing policy in the same install. The `setup_complexity` scoring axis should reflect this: `npx ruflo@latest init` masks substantial configuration surface area and partial-federation completion. The incomplete rebranding and the federation issue (#1669) are concrete quality signals that affect `governance_need` profiles. Conversely, at 38k stars and 100k MAU the adoption signal is strong enough that a clawfit recommendation engine that ignores it will produce credibility gaps with teams already using it.

## Preliminary interpretation
Current best reading:
- **Level 2 — Meta wrappers / harnesses / orchestration layers** (primary — explicit harness design: hooks into Claude Code lifecycle, queen-agent coordination, topology management, tiered routing, background workers, and the GOAP goal planner all operate at the harness layer above the base runtime)
- **Level 5 — Memory / MCP / context layer** (secondary — AgentDB with HNSW vector indexing, SONA trajectory learning, ReasoningBank, and cross-session vector persistence via RVF format are a full memory substrate, not a lightweight plugin)
- **Level 4c — MCP / tool-use layer** (secondary — 210 MCP tools, 32 plugins, and browser-tool gallery; Ruflo acts as both MCP server and MCP client, which is a distinct adapter role from pure tool registries)
- **Level 1 — Base runtimes / primary agent surfaces** (weak tertiary — multi-provider routing in flo.ruv.io and the Ruflo CLI's ability to operate without Claude Code create a thin base-runtime surface; not the primary classification but worth noting for profiles that evaluate provider independence)

The multi-layer collapse (L2 + L5 + L4c) is more complete than any prior tracked harness except Warp (L6 + L2 + L1). The L5 component is structurally larger here than in Warp.

Regarding reference-levels.md: this signal is strong enough that the "batteries-included swarm harness" pattern deserves a named sub-type entry in L2, distinct from (a) minimal harnesses (Pu.sh, mini-swe-agent), (b) methodology-SSOT harnesses (gsd, obra/superpowers), and (c) sprint-contract / single-cycle harnesses (Anthropic sprint-contracts, Hashline). Ruflo would be the canonical named example of a **swarm-orchestration harness** sub-type alongside deepagents (production-SDK harness). Do NOT add to reference-levels.md yet — the incomplete federation implementation and unverified benchmark claims require at least one independent technical audit or a v4 stable release before promotion. Flag for re-evaluation at v4.0 or confirmed 50k stars.
