# Research Watch: Agentic AI Foundation — MCP, goose, AGENTS.md become foundation-backed standards

- Agentic AI Foundation: <https://eu.36kr.com/en/p/3589328631202050>
- MCP Roadmap 2026: <https://thenewstack.io/model-context-protocol-roadmap-2026/>

## Why this is worth watching
Microsoft, Google, OpenAI, Anthropic, and the Linux Foundation have jointly created the **Agentic AI Foundation** to govern open-source agent standards. This is a structural governance shift — the same transition that defined how HTTP, containers, and Kubernetes went from project to standard. Three projects are at the core:
- **MCP** (Anthropic) — integration protocol, already at 97M monthly SDK downloads
- **Block's goose** — agent framework
- **OpenAI AGENTS.md** — agent specification format

## What stands out immediately
- MCP donated to the foundation in December 2025 — moved from Anthropic-controlled to multi-stakeholder governance
- 97 million monthly SDK downloads (Python + TypeScript) — MCP is already infrastructure-scale
- All major AI labs co-founding = unusual alignment; typically competitors avoid shared governance
- **AGENTS.md** is a new specification format from OpenAI — parallels to CLAUDE.md but as a cross-platform standard
- 2026 roadmap: clearer decision-making structures, contributor pathways, consistent integration patterns

## Why clawfit should care
MCP is already in clawfit's Level 4 as a capability standard. The governance shift changes its weight in the taxonomy — it is moving from "Anthropic's protocol" to "the dominant interoperability layer for agent tooling." This affects:
1. Any new tool claiming MCP compatibility should now be treated as aligning with a foundation standard, not just a single vendor
2. **AGENTS.md** is a new axis: if cross-platform agent spec format gains adoption, it competes with repo-specific CLAUDE.md patterns
3. goose (Block) is already tracked at Level 1 — its elevation to foundation project increases its strategic weight

## Preliminary interpretation
- Governance shift: **cross-cutting** — affects Level 2 (harnesses that implement MCP), Level 4 (MCP tools), Level 3 (AGENTS.md as executable SSOT)
- AGENTS.md: **Level 3 — Team harness / executable SSOT** (if adopted as cross-platform standard)
- Foundation itself: structural context, not a product layer

## Status
- Critical governance signal. Should be noted in reference-levels.md Level 4c and Level 3.
- AGENTS.md warrants its own future research-watch once spec content is public.
