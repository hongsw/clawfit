# Introducing clawfit

`clawfit` started from a simple question:

**How do we compare AI agents meaningfully?**

At first, this looked like a question about coding agents.
Then it became a question about orchestration.
Very quickly, it became obvious that this was still too narrow.

The AI agent ecosystem is not just a list of tools.
It is splitting into layers.

That is what `clawfit` is trying to map.

---

## What clawfit is

`clawfit` is a public, evolving map of the AI agent ecosystem.

It is being built to track more than just “which coding agent is best.”
It tries to map the structural layers that are forming around modern agent systems, including:

- base runtimes / primary agent surfaces
- meta wrappers / harnesses / orchestration layers
- team workflow / executable SSOT systems
- capability extensions (MCP, memory, tooling, plugins)
- research / evaluation / benchmark patterns
- data / evidence / knowledge infrastructure
- human interface / voice / talk-mode layers
- and now, methodology signals such as vibe-coding, harness engineering, and vibe-physics

In short:

> `clawfit` is not just tracking tools.
> It is tracking how the ecosystem is structurally changing.

---

## Why this exists

Most discussions about AI agents are still too flat.

They collapse very different things into one bucket:
- coding assistants
- orchestration systems
- wrappers
- memory layers
- team workflow packs
- research agents
- voice interfaces
- methodology shifts

That flattening makes it hard to see what is actually happening.

`clawfit` exists to make those differences legible.

---

## What clawfit is building toward

Right now, `clawfit` combines:
- a layered ecosystem reference model
- trend scans
- research watch notes
- reference notes
- methodology signal tracking
- analysis posts about emerging patterns

The goal is to turn that into a living map that helps answer questions like:
- what counts as a base runtime vs a harness?
- where do team workflow systems fit?
- how do MCP and memory layers reshape the landscape?
- why are voice/talk-mode systems becoming strategically important?
- when is a new repo a tool, a wrapper, a capability layer, or a methodology signal?

---

## Current thesis

The strongest current `clawfit` thesis is this:

> the future is not just “better coding agents.”
> it is a layered ecosystem of runtimes, harnesses, workflows, capabilities, research systems, interfaces, and methodologies.

That is why `clawfit` needs a bigger map.

---

## Who this is for

`clawfit` is useful if you are trying to understand:
- where AI coding is going
- how agent orchestration is evolving
- why harness layers are appearing
- how team workflow systems are becoming real infrastructure
- where voice/talk-mode interfaces may commercialize first
- how methodology itself is becoming a first-class axis
