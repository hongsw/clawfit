# Methodology Signals

This folder tracks **AI-native build methodologies** and creation paradigms.

These are not always products or runtimes.
They are often ways of working that shape how people:
- explore problems
- structure collaboration with AI
- turn prompting into repeatable research or build processes
- evolve from tool usage into methodology usage

Examples:
- vibe-coding
- harness engineering
- executable SSOT
- collaborative learning
- vibe-physics
- spec-driven development

Use this folder for signals where the important shift is not just *what tool exists*, but *how work is being done*.
