# LinkedIn intro draft

I’ve been working on a project called **clawfit**.

It started from a simple question:
**How do we compare AI agents meaningfully?**

Very quickly, it became obvious that the answer is bigger than “which coding agent is best.”

The ecosystem is splitting into layers:
- base runtimes
- harness / wrapper systems
- team workflow / executable SSOT layers
- capability extensions like MCP, memory, and tools
- research / benchmark patterns
- voice / talk-mode interfaces
- and now even methodology signals such as vibe-coding, harness engineering, and vibe-physics

So `clawfit` is becoming a public map of that ecosystem.

Not just a repo list.
A layered ecosystem map.

If you want to explore it:
- Main map: https://github.com/hongsw/clawfit/blob/main/docs/reference-levels.md
- Intro post: https://github.com/hongsw/clawfit/blob/main/docs/posts/2026-03-28-introducing-clawfit.md
- Overview hub: https://github.com/hongsw/clawfit/blob/main/docs/pages/ecosystem-overview.md

I’m especially interested in where this goes next around harness layers, team workflow systems, and AI-native methodologies.
